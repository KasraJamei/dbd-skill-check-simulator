# DBD Skill Check Simulator

A high-fidelity, timing-focused Dead by Daylight style skill check simulator built with Vite, React, TypeScript, TailwindCSS, Zustand, HTML5 Canvas, requestAnimationFrame, Web Audio API, and Tauri.

The simulator is designed from the ground up around ultra-low input latency: the canvas animation loop entirely owns needle motion, keyboard input is captured via direct global hooks outside the React render flow, and hit detection scoring is computed with high-resolution `performance.now()` microsecond timestamps.

## Features

- **Multiple Game Modes:** Standard, Decisive Strike, Hex: Ruin, Chaos, and a custom Zen Trainer mode.
- **Microsecond Precision:** Delta-time based `requestAnimationFrame` loop for silky smooth 60fps+ motion.
- **Zero-Lag Inputs:** Keyboard-first input with fully configurable keybinds bypasses React's virtual DOM engine.
- **Dynamic Audio Integration:** Adaptive Web Audio architecture playing custom audio cues or clean low-level fallback synthesizer tones.
- **Detailed Analytics:** Track score, streaks, perfect hits, raw milliseconds of reaction window, and per-mode metrics.
- **Local Leaderboards:** Timed survival runs that automatically save top records locally.
- **Fully Standalone Desktop Builds:** Optimized cross-platform desktop distributions for Windows and Linux powered by Tauri.

## Technical Architecture

- `src/components` — React UI shells, configuration menus, and the core Canvas mounting node.
- `src/modes` — Configuration tuning and random algorithmic attempt generators per game rules.
- `src/systems` — Main animation system loops, custom keydown adapters, and hit evaluation algorithms.
- `src/store` — Persistent client states using Zustand and `localStorage` middleware.
- `src/audio` — Timing cue managers handling standard synthetic sound waves and custom local buffers.
- `src/utils` — Pure trigonometric mapping functions, layout helpers, and timestamp validators.

> **Design Paradigm:** React handles static settings layout and numeric display panels only. The active timing system renders direct raster shapes straight to a Canvas element, keeping visual refresh routines isolated from frame-dropping React re-renders.

Skill checks spawn at random positions inside the play box. The hit zone is generated by travel distance from the needle start, so the check does not appear immediately beside the starting point. The timing guide setting draws subtle boundary lines at the success-zone edges. Zen Trainer enables it automatically.

## Installation & Setup

### Development Server

bash
# Install dependencies
npm install

# Start local web development server
npm run dev

The client app will open at `http://127.0.0.1:1420` (default port mapped for native webview proxy tracking).

### Production Web Build

bash
# Compile and optimize frontend distribution bundles
npm run build

# Preview production bundles locally
npm run preview

### Desktop Distribution Builds

bash
# Invoke Tauri compiler layer to construct a native binary
npm run tauri build

Production installer assets are generated directly into `src-tauri/target/release/bundle/`.

## Custom Sound Effects

Official Dead by Daylight sound effects are not packaged due to copyright. To load custom sound profiles, drop matching files into your project directory before compiling or mounting:

- `public/audio/skill-check-warning.ogg` — Triggered when the warning sound pattern plays.
- `public/audio/skill-check-success.ogg` — Triggered when a normal success target area is checked.
- `public/audio/skill-check-great.ogg` — Triggered when a perfect white target zone is hit.
- `public/audio/skill-check-fail.ogg` — Triggered during a zone miss or needle timeout.

The engine automatically scans for these local routes. If missing, it immediately switches to native system synthesizers to guarantee an environment without game-breaking audio latency.
